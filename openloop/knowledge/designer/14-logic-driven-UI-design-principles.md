### 14 logic-driven UI design principles:

1. Space elements based on how closely related they are
Use spacing to show relationships between elements. More closely related elements should be closer together, while unrelated elements should have more space between them. Use predefined spacing options (like an 8-point grid) rather than arbitrary spacing.
2. Ensure interface elements have a 3:1 contrast ratio
UI components like buttons and form fields need at least a 3:1 contrast ratio to meet accessibility standards. Use tools to measure contrast and ensure people with vision impairments can see interface details clearly.
3. Use a single primary button for the most important action
Highlight only one most important action per screen with a primary button. Multiple primary buttons compete for attention and cause confusion about what users should do next.
4. Ensure buttons have a sufficient target size
Make buttons at least 48pt by 48pt in size, separate them by at least 8pt, and make frequently used buttons even larger. This improves usability, especially for users with motor impairments or using mobile devices.
5. Make sure important content is visible
Don't hide crucial content behind menus if there's space to display it. People don't use what they can't see, so prioritize making important actions and information visible when needed.
6. Decrease letter spacing for large text
Reduce letter spacing on large headings to improve aesthetics. Many typefaces designed for small text have wide letter spacing that looks awkward at larger sizes.
7. Don't rely on color alone as an indicator
Use additional visual cues beyond color to convey meaning. Add underlines, icons, or other visual elements to ensure information is accessible to users with color blindness or low vision.
8. Try to avoid using multiple alignments
Stick to as few alignment types as possible (left, right, center). Multiple alignments within a small area create visual complexity and increase cognitive load.
9. Ensure text has a 4.5:1 contrast ratio
Small text needs at least 4.5:1 contrast ratio, while large text needs at least 3:1. This ensures readability for users with vision impairments.
10. Consider removing containers to simplify an interface
Group related elements using spacing, similarity, and alignment rather than always adding containers. Containers can add unnecessary visual clutter when other grouping methods work just as well.
11. Use regular and bold font weights only
Limit yourself to regular and bold font weights to maintain simplicity and consistency. Too many font weights add noise and make consistent usage more difficult.
12. Be consistent
Ensure similar elements look and work similarly throughout the interface. This includes visual consistency (rounded corners, spacing) and functional consistency with established patterns.
13. Don't confuse minimalism with simplicity
Minimal doesn't always mean simple or usable. Don't remove critical information in pursuit of a clean aesthetic. Add necessary details like text labels to ensure clarity and accessibility.
14. Balance icon and text pairs
When pairing icons with text, ensure they have similar visual prominence. Adjust color, size, or weight so neither element dominates the other for a more balanced appearance.
These principles focus on logic and accessibility rather than subjective aesthetic choices, making them practical guidelines for improving any interface design.
